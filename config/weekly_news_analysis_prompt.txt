## 出力形式

### 1. 週間サマリー
この週に目立った技術テーマ、媒体の関心、週全体として見えた変化を通常テキストで簡潔にまとめてください。
単日だけの記事を週次トレンドとして過大評価せず、複数日または複数記事で確認できる動きを優先してください。

### 2. 技術カテゴリ別の週間件数
日次レポートの技術カテゴリ別件数を合算し、以下のTSV形式で出力してください。

```tsv
技術カテゴリ	週間件数	主な掲載日	代表記事
生成AI	数値のみ	日付または不明	代表記事がなければ不明
セキュリティ	数値のみ	日付または不明	代表記事がなければ不明
クラウド	数値のみ	日付または不明	代表記事がなければ不明
開発	数値のみ	日付または不明	代表記事がなければ不明
その他	数値のみ	日付または不明	代表記事がなければ不明
```

### 3. 週間の注目テーマ
日次分析で繰り返し現れた、または週後半に強まったテーマを最大5件抽出し、以下のTSV形式で出力してください。
根拠は、日付・カテゴリ・代表記事から追えるようにしてください。

```tsv
注目テーマ	技術カテゴリ	週内での動き	根拠日付	代表記事	広報上の示唆
テーマ名	カテゴリ名	継続または後半増加または単発	日付	記事タイトル	短い示唆
```

### 4. 日次から見た変化点
週の前半・中盤・後半で、記事の論点や媒体の関心がどう変化したかを通常テキストで分析してください。
日次レポートに書かれていない事実は断定しないでください。

### 5. 翌週に向けた広報の切り口
この週の傾向を踏まえ、翌週に狙うべき広報テーマを3つ、通常テキストで具体的に提示してください。

## TSV出力ルール
- 表はすべて fenced code block 内に `tsv` として出力してください。
- 1行目は必ずヘッダー行にしてください。
- 区切り文字はタブのみを使ってください。
- Markdown表の `|`、区切り罫線、箇条書き記号を表内に入れないでください。
- セル内に改行を入れないでください。
- 件数は `3` のような数値だけにし、`3件` とは書かないでください。
- URLが必要な場合は1セルにそのまま入れてください。
- 不明な値は空欄ではなく `不明` と書いてください。
- 一時的な単発記事と、複数日にまたがる継続トレンドを明確に区別してください。
